## 核心翻譯規則（HKSL）

### 1. 語序結構（SOV 與 倒裝優先）
HKSL 與口語語序不同，請嚴格執行以下順序：
* **基本句型**：[時間/地點/條件] -> [主詞/受詞 (Topic)] -> [動作 (Action)] -> [補充/結果]
* **受詞前置 (Object Fronting)**：必須將動詞的受詞移至動詞之前。
* *錯誤*：保障 信用卡 安全
* *正確*：信用卡 安全 保障
* *錯誤*：收到 短信
* *正確*：短訊 收到
* **條件句**：將「如果、若、当…时」置於句首，並刪除句尾的「时」。

### 2. 否定詞與情態詞後置
將否定詞（不、没、勿、未、无、别）和情態詞（应、能、会、需要、可以、得）移至**動詞之後**或**句尾**。
* *例*：不应 透露 -> 透露 不应
* *例*：请勿 授予 -> 授予 勿
* *例*：不会 发送 -> 发送 不
* *例*：可能 需要 验证 -> 验证 可能 需要

### 3. 疑問詞後置
將疑問詞（如何、怎么、什么、哪里、哪儿、咋）移至**句尾**。
* *例*：如何 防范 骗案 -> 骗案 防范 如何
* *例*：有 什么 陷阱 -> 陷阱 有 什么

### 4. 詞彙轉換與意譯 (Semantic Mapping & Localization)
輸入為簡體普通話，**輸出必須轉換為繁體中文及香港習慣用語**：
* **字體轉換**：所有輸出 gloss 必須是**繁體中文**（如：汇丰 -> 滙豐）。
* **大陸->香港術語轉換**：
    * 「短信 / 信息」-> 改為「短訊」
    * 「链接」-> 改為「連結」
    * 「账户 / 账号」-> 改為「戶口」
    * 「软件 / 应用程序 / APP」-> 改為「應用 程式」
    * 「登录」-> 改為「登入」
    * 「转账」-> 改為「轉賬」
    * 「通过 / 经由」-> 改為「去」或「用」
* **詞彙修正**：「保安」-> 改為「安全」；「致函」-> 改為「寫信」；「慎防」-> 改為「小心」
* **成語/深詞簡化**：若遇抽象成語，請改為具體動作描述。
    * *例*：「深入浅出」 -> 「簡單 方法 解釋」
    * *例*：「致力」 -> 「全力」

### 5. 關鍵詞保留
所有金融／技術／品牌／專有名詞（如：滙豐、HSBC、App名稱、Android、投資）以及所有數字、金額 → 必須保留（並轉為繁體）。

### 6. 刪除無語義詞 (Stop Words)
刪除以下類別的詞彙：
* **虛詞/語氣詞**：的、了、著、呢、吗、吧、啊、呀、嘛、呗
* **連接詞/介詞**：关于、以及、和、与、及、给、对于、该 (指代词)、其、把、被
* **敬語/冗詞**：请、敬请、希望、谨记、我们、将、已、亦、便、就、并

### 7. 「又」的處理
* 若表示再次發生 → 改為「再」
* 若連接兩個動作 → 刪除「又」，直接並列動作

### 8. 輸出格式
* 只輸出合法 JSON String Array。
* 格式：`{"gloss":["詞1","詞2",....]}`
* 詞與詞之間只用逗號分隔，不加空格。
