## 核心翻譯規則（HKSL）

### 1. 語序結構（SOV 與 倒裝優先）
HKSL 與口語語序不同，請嚴格執行以下順序：
* **基本句型**：[時間/地點/條件] -> [主詞/受詞 (Topic)] -> [動作 (Action)] -> [補充/結果]
* **受詞前置 (Object Fronting)**：必須將動詞的受詞移至動詞之前。
* *錯誤*：保障 信用卡 安全
* *正確*：信用卡 安全 保障
* *錯誤*：收到 短訊
* *正確*：短訊 收到
* **條件句**：將「如果、若、當…時」置於句首，並刪除句尾的「時」。

### 2. 否定詞與情態詞後置
將否定詞（不、沒、勿、未、無）和情態詞（應、能、會、需要、可以）移至**動詞之後**或**句尾**。
* *例*：不應 透露 -> 透露 不應
* *例*：請勿 授予 -> 授予 勿
* *例*：不會 發送 -> 發送 不
* *例*：可能 需要 驗證 -> 驗證 可能 需要

### 3. 疑問詞後置
將疑問詞（如何、怎樣、什麼、哪裡）移至**句尾**。
* *例*：如何 防範 騙案 -> 騙案 防範 如何
* *例*：有 甚麼 陷阱 -> 陷阱 有 什麼

### 4. 詞彙轉換與意譯 (Semantic Mapping)
雖然原則上保留原詞，但必須執行以下特定的語義轉換（基於習慣用法）：
* **介詞轉換**：「透過 / 經由」-> 改為「去」或「用」
* **詞彙修正**：「保安」-> 改為「安全」；「怎樣」-> 改為「如何」；「致函」-> 改為「寫信」；「慎防」-> 改為「小心」
* **成語/深詞簡化**：若遇抽象成語，請改為具體動作描述。
* *例*：「深入淺出」 -> 「簡單 方法 解釋」
* *例*：「致力」 -> 「全力」

### 5. 關鍵詞保留
所有金融／技術／品牌／專有名詞（如：滙豐、HSBC、App名稱、Android、投資）以及所有數字、金額 → 必須原樣保留。

### 6. 刪除無語義詞 (Stop Words)
刪除以下類別的詞彙：
* **虛詞/語氣詞**：的、了、著、呢、嗎、吧、啊、呀、喎
* **連接詞/介詞**：關於、以及、和、與、及、給、對於、該 (指代詞)、其
* **敬語/冗詞**：請、敬請、希望、謹記、我們、將、已、亦、便、就、並

### 7. 「又」的處理
* 若表示再次發生 → 改為「再」
* 若連接兩個動作 → 刪除「又」，直接並列動作

### 8. 輸出格式
* 只輸出合法 JSON String Array。
* 格式：`{"gloss":["詞1","詞2",....]}`
* 詞與詞之間只用逗號分隔，不加空格。
